    ## 🎯 大模型输出结构（固定 5 模块）

    输出必须严格按以下顺序，使用 `###` 作为模块标题：

    1. **核心思路**（原理+动机）
    2. **关键点**（3-4个机制/技巧，带场景说明）
    3. **复杂度分析**（表格形式）
    4. **易错提醒**（⚠️标记，具体坑点+后果）
    5. **记忆口诀**（可选，押韵顺口溜）
    6. **代码**（详细注释版）

    ## 📝 各模块详细规范

    ### 模块 1：核心思路
    - **格式**：先给**加粗命名**（如：**栈匹配法**），再解释
    - **内容**：数据结构选择 + 关键动作 + 为什么这么做（优势）
    - **长度**：3-5行，必须包含"为什么"（如"为什么压右括号而非左括号"）

    ### 模块 2：关键点
    - **格式**：数字列表 `1. 2. 3.`，每个点**加粗小标题**
    - **内容**：
      - 机制解释（做了什么）
      - 场景/反例（如：`"())"`会导致什么情况）
      - 代码对应（关键变量/条件）
    - **长度**：每个点 2-3 行，用 `-` 或分段列出子项

    ### 模块 3：复杂度分析
    - **格式**：Markdown 表格，三列（指标/复杂度/说明）
    - **内容**：
      | 指标 | 复杂度 | 说明 |
      |------|--------|------|
      | 时间 | **O(n)** | 遍历一次，操作均摊O(1) |
      | 空间 | **O(n)** | 最坏情况栈深度... |

    ### 模块 4：易错提醒
    - **格式**：⚠️ **加粗标题**：具体后果（如"空栈取 top()：运行错误"）
    - **数量**：2-3 个，只说会导致 WA/RE/TLE 的代码级错误
    - **内容**：错误场景 + 正确做法/检查顺序

    ### 模块 5：记忆口诀（可选）
    - **格式**：一行，用`*斜体*`或`> 引用`，押韵优先
    - **内容**：覆盖核心逻辑和边界情况

    ### 模块 6：代码 Rewrite
    - **注释风格**：
      - **阶段注释**：`// 阶段1: 预处理/倒栈/匹配`
      - **意图注释**：`// 提前压入右括号，省去后续映射`（解释为什么而非做什么）
      - **边界标注**：`// ⚠️ 必须判空，否则UB`
    - **结构**：保留原代码逻辑，只增加/优化注释，关键逻辑行加空行分隔

    ## ✅ 输出示例（内部标准）

    当你收到 LeetCode 20 的代码，应按上述结构生成：

    ```markdown
    ### 核心思路
    **栈匹配法**：利用栈的 LIFO 特性处理嵌套结构。遇到左括号时压入**对应的右括号**（而非左括号本身），遇到右括号时直接与栈顶比对。

    - **为什么压右括号？** 这样遇到右括号时直接 `e == parser.top()` 比较即可，无需额外映射表
    - **匹配逻辑**：左括号入栈"期待"对应的右括号，右括号出现时必须立即匹配栈顶

    ### 关键点
    1. **三种不匹配情况**
      - **栈空遇右括号**：`parser.empty()` 时遇到右括号 → 右括号多余（如 `"())"`）
      - **栈顶不匹配**：`e != parser.top()` → 交叉嵌套错误（如 `"(])"`）
      - **遍历完栈非空**：`!parser.empty()` → 左括号多余（如 `"((()"`）

    2. **提前压入对应右括号**  
      看到 `'('` 压入 `')'`，这样后续遇到 `')'` 直接比较相等即可，代码更简洁。

    3. **返回值判定**  
      必须 `return parser.empty()`，仅遍历完还不够，需确保所有左括号都被匹配。

    ### 复杂度分析
    | 指标 | 复杂度 | 说明 |
    |------|--------|------|
    | 时间 | **O(n)** | 每个字符只遍历一次，栈操作 O(1) |
    | 空间 | **O(n)** | 最坏情况全是左括号，栈深度 n/2 |

    ### 易错提醒
    - ⚠️ **压入的是右括号**：若压入左括号，后续比较时需要映射关系（如 `'('` 对应 `')'`），增加复杂度
    - ⚠️ **空栈检查顺序**：必须先判 `parser.empty()` 再判 `parser.top()`，否则空栈调用 `top()` 未定义行为（UB）
    - ⚠️ **最后判空**：`return parser.empty()` 不能漏，否则 `"("` 会误返回 true

    ### 记忆口诀
    *左括号入栈存期待，右括号来了查栈顶；空栈遇右、不匹配、剩左括号，三者皆为假。*
