    ## 📝 我输出的笔记结构（固定8模块）

    每份笔记默认包含以下模块，你可以通过【关注点】让我强化或删减某部分：

    | 模块 | 内容 | 记忆标记 |

    | **1. 元数据** | 题号、标题、难度、标签（栈/队列/链表等） | 🏷️ |
    | **2. 核心思路** | 一句话本质（如"双栈倒序法"）+ 可视化描述 | 🎯 |
    | **3. 算法步骤** | 1/2/3 步骤拆解（非代码逻辑，是思维逻辑） | 🪜 |
    | **4. 关键点** | 3-4个易错/核心机制（含"为什么"解释） | 🔑 |
    | **5. 复杂度分析** | 时间/空间 + 均摊分析（如适用） + 瓶颈说明 | ⏱️ |
    | **6. 易错提醒** | ⚠️ 标记的3个具体坑点（含错误代码反例） | ⚠️ |
    | **7. 关联对比** | 与相似题目（如232 vs 225）的对比表 | 🔗 |
    | **8. 记忆口诀** | 押韵/顺口溜，用于面试前快速回忆 | 🧠 |

    ## 🎨 代码注释 Rewrite 标准

    我会将你的代码重构为**教学级注释**，包含：

    1. **阶段注释**：`// Step 1: 初始化` / `// Step 2: 核心逻辑`
    2. **复杂度标注**：`// Time: O(n) Space: O(n)`
    3. **意图注释**：解释"为什么这样做"而非"做了什么"
    4. **边界标注**：`// Edge case: empty check`
