## 核心思路（精简）

**单调递减队列（Monotonic Deque）**：维护一个存储下标的**双端队列**，保证队列对应的数组值单调递减，队首始终是当前窗口最大值。

- **为什么是递减？** 新元素入队时，所有比它小的旧元素都"永无出头之日"（被新元素遮挡且新元素更晚过期），可直接从队尾剔除；这样队首永远是当前窗口可见的最大值
- **为什么存下标而非值？** 需要判断元素是否滑出窗口左边界（通过下标范围 `i-k+1`），仅存值无法判断"过期"

**关键点**

1. **窗口过期清理（队首弹出）**
   - 场景：窗口右滑时，左边界为 `i-k+1`，**队首下标 `<` 左边界则已滑出窗口**
   - 反例：若不及时清理，过期最大值会误导后续结果（如窗口 `[1,3,-1]` 右滑后，-1若未清掉1，会影响下一个窗口判断）
   - 代码对应：`if (!q.empty() && q.front() < i-k+1) q.pop_front();`

2. **维护单调递减性（队尾剔除）**
   - 机制：新元素入队前，从**队尾**弹出所有值 `≤` 新元素的旧下标
   - **为什么剔除 ≤？** 这些元素在窗口期内永远不可能成为最大值（被后面更大或相等但更新的元素"压制"且后者更晚过期）
   - 场景：`nums=[1,3,1,2]`，当 `i=1`（值3）入队时，队列中的下标0（值1）被弹出，因为只要3在窗口内，1永远不可能是最大值

3. **窗口形成判定**
   - 条件：`if (i < k-1) continue;`
   - 场景：前 `k-1` 个元素（下标 `0` 到 `k-2`）时窗口未满，不应记录结果
   - 首个记录点：`i=k-1` 时窗口 `[0..k-1]` 形成，此时 `q.front()` 即为第一个窗口最大值

4. **下标与值的映射关系**
   - 队列存储的是**下标**，比较数值时用 `nums[q.back()] <= nums[i]`
   - 记录结果时必须映射回数值 `nums[q.front()]`
   - 易混淆点：切勿直接用队列元素（下标）作为最大值输出，或混淆比较对象


**易错提醒**
- ⚠️ **剔除条件用 `<=` 而非 `<`**：相等的旧元素应被剔除，因为新元素下标更大、过期更晚，保留新元素更优；若保留旧相等元素，虽数值结果正确但逻辑错误且浪费空间
- ⚠️ **窗口边界计算错误**：左边界是 `i-k+1` 而非 `i-k`，队首 `<` 左边界即过期（如 `k=3, i=3` 时，窗口应为 `[1,2,3]`，左边界=1，下标0需弹出）


**复杂度分析**
- 空间复杂度：O(k)
- 时间复杂度：O(n) 相比暴力算法的O(kn)优化很多

## 代码实现

```cpp
class Solution {
public:
    vector<int> maxSlidingWindow(vector<int>& nums, int k) {
        deque<int> q;        // 阶段1: 初始化单调队列（存下标）
        vector<int> records; // 存储各窗口最大值结果
        
        // 阶段2: 遍历每个元素，维护队列
        for (int i = 0; i != nums.size(); ++i) {
            
            // 步骤2.1: 清理队首过期元素（滑出窗口左边界）
            // 左边界为 i-k+1，队首下标 < 左边界则已过期
            // ⚠️ 必须先清队首，再处理队尾和入队
            if (!q.empty() && q.front() < i - k + 1) {
                q.pop_front();
            }
            
            // 步骤2.2: 维护单调递减性（队尾剔除）
            // 意图: 新元素nums[i]入队前，所有≤它的旧元素永无出头之日，直接剔除
            // 为什么while? 可能连续多个元素都比新元素小
            while (!q.empty() && nums[q.back()] <= nums[i]) {
                q.pop_back();
            }
            
            // 步骤2.3: 当前索引入队
            q.push_back(i);
            
            // 步骤2.4: 窗口未形成时跳过记录
            // 场景: 前k-1个元素时窗口未满，不记录结果
            if (i < (k - 1)) continue;
            
            // 步骤2.5: 记录当前窗口最大值
            // 意图: 队首是队列中最大元素的下标，映射回nums取值
            records.push_back(nums[q.front()]);
        }
        
        return records;
    }
};
```
