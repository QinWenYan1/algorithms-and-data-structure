## 关键点（精简）

**为什么三种迭代方法没有统一？**
- 在**后序遍历** [binary_tree_2_postorder_traversal.md](./binary_tree_2_postorder_traversal.md)中，我们提到中提到说**使用栈的话，无法同时解决访问节点（遍历节点）和处理节点（将元素放进结果集）不一致的情况**。
- 实际上，迭代方法是可以统一地使用一种方法的！但我们需要将访问的节点放入栈中，**把要处理的节点也放入栈中但是要做标记**。

**那么如何统一迭代方法？**
- **方法一**：就是要处理的节点放入栈之后，紧接着放入一个空指针作为标记。 这种方法可以叫做**空指针标记法**。
    - 我们将访问的节点直接加入到栈中，但如果是处理的节点则后面放入一个空节点， 这样只有空节点弹出的时候，才将下一个节点放进结果集。
    - **空指针法**：节点只出现一次，但前面插了一个 `nullptr` 做"红绿灯"，**nullptr 被弹出 = 绿灯亮，下一个节点可以通行（输出）**。

    1. 第 1 点：核心思想：
        - 栈里不只存节点，还存 **nullptr（空指针）**。  
        - 这个 **nullptr 是一个"通知单"**——当它被弹出栈时，意味着：**"它后面紧跟着的那个节点，左右子树已经全部处理完毕，现在可以直接输出它了。"**


    2. 第 3 点：遇到非空节点怎么办？（最关键）：
        - 当弹出的是一个**非空节点**时，**绝对不能现在输出它**（后序要最后才访问根）。  
        - 我们要把它按特定顺序"塞回"栈里，让它等待左右子树先处理完。
        - 因为栈是**后进先出**，所以实际弹出顺序变成了：
            ```
            左子树 → 右子树 → nullptr → 节点本身
            ```

    3. 第 4 点：遇到 `nullptr` 怎么办？：
        - 当弹出的是 **`nullptr`** 时，说明左右子树都处理完了。  
        - 此时栈顶紧接着就是等待输出的那个节点，直接再 pop 一次，把值加入结果。

    4. 第 5 点：叶子节点会怎么走？：
        - 叶子没有左右孩子，压栈后栈里只剩 `[node, nullptr]`。  
        - 下一次遇到 None，直接输出该叶子。完美符合"左右都为空，输出根"的逻辑。

用这棵树：

```
        1
       / \
      2   3
     /
```

**后序遍历正确结果：4 → 2 → 3 → 1**

- 下面我一步一步走栈（栈底 → 栈顶）：

    | 步骤 | 栈内状态（底 → 顶） | 弹出的元素 | 操作说明 | 结果 |
    |------|---------------------|------------|----------|------|
    | 初始 | `[1]` | — | 根节点入栈 | `[]` |
    | 1 | `[1, nullptr, 3, 2]` | `1` | 弹出 1（非空），按顺序压回：`1`→`nullptr`→`3`→`2` | `[]` |
    | 2 | `[1, nullptr, 3, 2, nullptr, 4]` | `2` | 弹出 2（非空），压回：`2`→`nullptr`→`4` | `[]` |
    | 3 | `[1, nullptr, 3, 2, nullptr, 4, nullptr]` | `4` | 弹出 4（非空，叶子），压回：`4`→`nullptr` | `[]` |
    | 4 | `[1, nullptr, 3, 2, nullptr]` | `nullptr` | 弹出 nullptr！再弹出下一个 `4`，输出 | `[4]` |
    | 5 | `[1, nullptr, 3]` | `nullptr` | 弹出 nullptr！再弹出下一个 `2`，输出 | `[4, 2]` |
    | 6 | `[1, nullptr, 3, nullptr]` | `3` | 弹出 3（非空，叶子），压回：`3`→`nullptr` | `[4, 2]` |
    | 7 | `[1, nullptr]` | `nullptr` | 弹出 nullptr！再弹出下一个 `3`，输出 | `[4, 2, 3]` |
    | 8 | `[]` | `nullptr` | 弹出 None！再弹出下一个 `1`，输出 | `[4, 2, 3, 1]` |
