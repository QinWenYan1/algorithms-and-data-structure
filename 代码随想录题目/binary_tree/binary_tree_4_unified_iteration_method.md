## 关键点（精简）

**为什么三种迭代方法没有统一？**
- 在**后序遍历** [binary_tree_2_postorder_traversal.md](./binary_tree_2_postorder_traversal.md)中，我们提到中提到说**使用栈的话，无法同时解决访问节点（遍历节点）和处理节点（将元素放进结果集）不一致的情况**。
- 实际上，迭代方法是可以统一地使用一种方法的！但我们需要将访问的节点放入栈中，**把要处理的节点也放入栈中但是要做标记**。

**那么如何统一迭代方法？**
- **方法一**：[🔗 代码随想录](https://programmercarl.com/%E4%BA%8C%E5%8F%89%E6%A0%91%E7%9A%84%E7%BB%9F%E4%B8%80%E8%BF%AD%E4%BB%A3%E6%B3%95.html#%E6%80%9D%E8%B7%AF)  

- **方法二**：就是要处理的节点放入栈之后，紧接着放入一个空指针作为标记。 这种方法可以叫做**空指针标记法**。
    - 我们将访问的节点直接加入到栈中，但如果是处理的节点则后面放入一个空节点， 这样只有空节点弹出的时候，才将下一个节点放进结果集。
    - **空指针法**：节点只出现一次，但前面插了一个 `nullptr` 做"红绿灯"，**nullptr 被弹出 = 绿灯亮，下一个节点可以通行（输出）**。
    - **遇到非空节点怎么办？**（最关键）：
        - 当弹出的是一个**非空节点**时，**绝对不能现在输出它**（后序要最后才访问根）。  
        - 我们要把它按特定顺序"塞回"栈里，让它等待左右子树先处理完。

**以中序遍历为例 -- 空指针标记法**

1. **遇到普通节点：逆序压栈控制遍历序**  
   - **机制**：弹出普通节点后，按 **右子树 → 节点本身 → `nullptr` → 左子树** 的顺序压回栈；其中压入左右子树前均显式检查 `if (curr->right) `和 `if (curr->left)`，空指针直接跳过不入栈
    - **代码意图**：栈后进先出，保证下一轮实际弹出顺序为 **左子树 → `nullptr` → 节点 → 右子树**；过滤空子树可防止将 `nullptr` 当作"待遍历子树"压入栈，避免后续循环中将其误判为哨兵或导致空指针解引用

3. **遇到哨兵 `nullptr`：直接输出下一个节点**  
   - **机制**：`if (s.top())` 为假时进入 `else` 分支，先弹出哨兵，再弹出栈顶节点加入结果  

4. **叶子节点的天然收敛**  
   - **场景**：叶子节点无左右孩子，拆解时仅压入 **自身 + `nullptr`**  
   - **机制**：下一轮循环栈顶即为 `nullptr`，立即触发输出分支，弹出哨兵后输出该叶子  

5. **循环分支的严格二分**  
   - **机制**：循环开头通过 `if (s.top())` 将流程严格分为"拆解节点"和"输出节点"两路，互不渗透  
   - **代码意图**：防止 `nullptr` 被当作普通节点进入子树拆解逻辑（这是初版代码死循环的根源）  
   - **终止条件**：栈空时循环结束，所有节点和哨兵均已配对弹出，结果序列完整，无残留状态

## 代码实现
```cpp

class Solution {
public:
    vector<int> inorderTraversal(TreeNode* root) {
        vector<int> result;
        stack<TreeNode*> st;
        
        if (root) st.push(root);
        
        while (!st.empty()) {
            if (st.top()) {
                // 第一次遇到该节点：按中序逆序压栈
                // 中序遍历顺序：左 -> 中 -> 右
                // 栈后进先出，所以压入顺序：右 -> 中 -> 哨兵 -> 左
                TreeNode* node = st.top();
                st.pop();
                
                if (node->right) st.push(node->right);  // 右子树（最先压，最后处理）
                st.push(node);                          // 节点本身
                st.push(nullptr);                       // 哨兵：左子树遍历完后输出该节点
                if (node->left)  st.push(node->left);   // 左子树（最后压，最先处理）
                
            } else {
                // 遇到哨兵：左子树已全部遍历，现在输出该节点
                st.pop();                               // 弹出哨兵
                result.push_back(st.top()->val);    // 输出节点
                st.pop();
            }
        }
        
        return result;
    }
};

```
