## 关键点（精简）

**基础概念**

- 概念与计算方向
    - **深度**：是从上到下计算的，指从根节点自顶向下到某个节点的路径长度。对于整棵树而言，其深度是指所有结点中最深的叶节点所在的层数。
        - **使用前序遍历，将计算从上往下传导**：从中间乡下面遍历一个字节点就 `depth + 1`
    - **高度**：是从下到上计算的，指从某个节点自底向上到最底层叶子节点的路径长度。对于整棵树而言，其高度是指根节点到最远叶子节点的路径长度。
        - **使用后序遍历，将计算从下往上传导**：计算左，计算右，完后再汇总到中通过 `max` 计算高度并向上传导 `depth + 1`
        - 实际上求高度 = 深度，本题通过求最高高度得到树的深度

**核心思路 -- 求最大高度**
- **分治递归法**：将"整棵树的最大深度"拆解为"左右子树最大深度的较大值 + 1"。递归终止于空节点，深度为 0。

    - **为什么是 +1 而非 +2？** 当前节点自身贡献一层深度，子树深度已包含其子节点层数，叠加时只需加当前这一层

1. **递归终止条件**
   - `root == nullptr` 返回 0，空树无节点，深度为 0
   - 若误返回 1，则单层节点树会算出深度为 2，导致系统性偏差

2. **分治合并逻辑**
   - 左子树深度 `maxDepth(root->left)` 与右子树深度 `maxDepth(root->right)` 取较大值

3. **后序遍历特性**
   - 必须先递归到左右子树底部拿到深度，再在当前层做 `max + 1` 合并
   - 代码结构上体现为 `return 1 + max(...)`，当前节点值在递归结果返回后才参与计算

- ⚠️ **空节点返回 0 而非 1**：若 `return 1` 当 `root == nullptr`，叶子节点的空子树会额外贡献一层，导致所有深度 +1
- ⚠️ **用 max 而非 min**：题目求最大深度，误写 `min` 会导致只取较浅分支，对非满二叉树返回错误结果
- ⚠️ **忘记 +1**：若直接 `return max(...)` 漏掉当前节点自身的一层，整棵树深度会少算 1

**复杂度分析**
| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | **O(n)** | 每个节点仅访问一次，递归调用常数开销 |
| 空间 | **O(h)** | 递归栈深度等于树高 h，最坏情况 h=n（链状树） |


**记忆口诀**：*空树深度零，左右取最大，加一归本层，递归到万家。*

**核心思路 -- 层序迭代法**
- 直接通过层序遍历即可，参考[文字解析（代码随想录）](https://programmercarl.com/0104.%E4%BA%8C%E5%8F%89%E6%A0%91%E7%9A%84%E6%9C%80%E5%A4%A7%E6%B7%B1%E5%BA%A6.html#%E7%AE%97%E6%B3%95%E5%85%AC%E5%BC%80%E8%AF%BE)

## 代码实现
```cpp
class Solution {
public:
    int maxDepth(TreeNode* root) {
        // 阶段1: 递归终止 —— 空节点无深度，返回 0
        if (root == nullptr) return 0;

        // 阶段2: 分治合并 —— 先递归求左右子树深度，取较大值后加当前层
        // 为什么是 max？题目要求最大深度，取 min 会漏掉更深分支
        // 为什么 +1？当前节点自身贡献一层深度
        return 1 + max(maxDepth(root->left), maxDepth(root->right));
    }
};
```
